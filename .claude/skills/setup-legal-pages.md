# 法務ページ整備 (利用規約 / プライバシー / 特商法)

zuk-zuk-template ベースの案件に **3 種類の法務ページ** を整備する Skill。
個人事業主・法人、サブスク有無、EC 有無 で必要項目が変わるので、案件タイプ別に分岐する。

## 必須となる 3 ページ

| ページ | パス | 必須案件 |
|---|---|---|
| **利用規約** | `/legal/terms` | サブスク / EC / 会員機能ある案件 |
| **プライバシーポリシー** | `/legal/privacy` | 個人情報を取得する全案件 (=ほぼ全て) |
| **特定商取引法に基づく表記** | `/legal/tokushoho` | 商品・サービスをオンライン販売する案件 |

> ⚠️ **個人情報保護法**は、フォームで個人情報を取得する全案件で対象。プライバシーポリシーは事実上必須。

## 前提条件

- ✅ zuk-zuk-template クローン済み
- ✅ クライアントが **個人事業主 / 法人 / 株式会社** どれか確定
- ✅ 適格請求書発行事業者番号 (T番号) の有無確認 (BtoB 取引するなら必須)
- ✅ 月額契約案件の場合、最低契約期間・解約条件が決定
- ✅ EC 案件の場合、商品・送料・返品条件が決定

## 手順

### Step 1. ディレクトリ作成

```
src/app/
└── legal/
    ├── terms/
    │   └── page.tsx          # 利用規約
    ├── privacy/
    │   └── page.tsx          # プライバシーポリシー
    └── tokushoho/
        └── page.tsx          # 特商法表記
```

### Step 2. 利用規約 (`/legal/terms`)

**必須項目** (サブスクの場合):

| 項目 | 内容 |
|---|---|
| 第 1 条 (適用) | 規約の適用範囲 |
| 第 2 条 (利用登録) | アカウント作成 / 申込み |
| 第 3 条 (利用料金・支払方法) | 月額料金、支払方法 (クレジットカード等) |
| 第 4 条 (禁止事項) | スパム、不正利用等 |
| 第 5 条 (本サービスの提供の停止等) | メンテ・障害時 |
| 第 6 条 (著作権) | サイトコンテンツの帰属 |
| 第 7 条 (利用制限および登録抹消) | 違反時の対応 |
| 第 8 条 (**最低契約期間と解約**) | サブスク特有: 12 ヶ月最低 / 中途解約手数料 |
| 第 9 条 (退会) | 解約手続き |
| 第 10 条 (保証の否認および免責事項) | 損害賠償の上限 |
| 第 11 条 (サービス内容の変更等) | 規約・料金改定 |
| 第 12 条 (利用規約の変更) | 通知方法 |
| 第 13 条 (個人情報の取扱い) | プライバシーポリシーへの誘導 |
| 第 14 条 (準拠法・裁判管轄) | 日本法・東京地裁等 |

サンプルテンプレ (`src/app/legal/terms/page.tsx`):

```tsx
import type { Metadata } from "next";
import { SITE } from "@/lib/content";

export const metadata: Metadata = {
  title: "利用規約",
  description: `${SITE.name} のサービス利用規約。`,
};

export default function TermsPage() {
  return (
    <main className="legal-page wrap">
      <h1 className="h-section">利用規約</h1>
      <p>本規約は、{SITE.name} (以下「当方」) が提供する...について定めるものです。</p>

      <h2>第 1 条 (適用)</h2>
      <p>...</p>

      <h2>第 2 条 (利用登録)</h2>
      <p>...</p>

      {/* ... 案件用に編集 ... */}

      <p style={{ marginTop: 64, color: "var(--fg-2)" }}>
        制定日: 2026 年 X 月 X 日
        <br />
        最終改訂日: 2026 年 X 月 X 日
      </p>
    </main>
  );
}
```

### Step 3. プライバシーポリシー (`/legal/privacy`)

**必須項目** (個人情報保護法対応):

| 項目 | 内容 |
|---|---|
| 1. 個人情報の定義 | 「個人情報」とは何か |
| 2. 個人情報の収集方法 | フォーム / Cookie / アクセス解析 |
| 3. 個人情報の利用目的 | サービス提供 / 連絡 / 改善 |
| 4. 個人情報の第三者提供 | 原則しない、例外 (法令等) |
| 5. 個人情報の安全管理 | 適切に管理する旨 |
| 6. 個人情報の開示・訂正・削除 | お客様の権利 + 連絡先 |
| 7. Cookie の使用 | 何のために使うか (GA4 / Search Console 等) |
| 8. アクセス解析ツール | Google Analytics 4 等 |
| 9. プライバシーポリシーの変更 | 通知方法 |
| 10. お問い合わせ窓口 | 個人情報取扱責任者 / 連絡先 |

### Step 4. 特定商取引法に基づく表記 (`/legal/tokushoho`)

**必須項目** (オンライン販売案件で必須):

| 項目 | 内容 |
|---|---|
| 販売事業者 (販売者) | 個人事業主: 氏名 / 法人: 会社名 |
| 運営責任者 | 個人: 同上 / 法人: 代表者氏名 |
| 所在地 | **個人事業主**: 「請求があった場合に遅滞なく開示」可 (自宅住所の公開回避) |
| 連絡先 (電話番号) | 同上、または「請求時開示」 |
| メールアドレス | 必須開示 |
| 適格請求書発行事業者番号 (T番号) | 該当する場合に開示 |
| 販売価格 | 個別商品 / 月額料金 等 |
| 商品代金以外の必要料金 | 送料 / 振込手数料 / 消費税 等 |
| 申込有効期限 | 該当する場合 |
| 引渡時期 / 提供開始時期 | 注文後 X 営業日 / 申込直後 等 |
| 支払方法 | クレジットカード / 銀行振込 等 |
| 支払時期 | 注文時 / 月末締め 等 |
| 返品・キャンセル | 不良品の場合 X 日以内 / デジタル商品はキャンセル不可 等 |
| **最低利用期間** (サブスク) | 12 ヶ月等 |
| **中途解約条件** (サブスク) | 残月分一括請求 等 |

> ⚠️ **個人事業主の特例**: 「所在地・電話番号」は、購入者から請求があった場合に遅滞なく開示すれば、ウェブ上に公開不要 (令和 4 年改正)。

### Step 5. Footer / Header にリンク追加

`src/lib/content.ts` の `FOOTER_COLUMNS` の Legal カラムは既にテンプレに含まれている:

```ts
{
  heading: "Legal",
  links: [
    { label: "プライバシーポリシー", href: "/legal/privacy" },
    { label: "利用規約", href: "/legal/terms" },
    { label: "特定商取引法に基づく表記", href: "/legal/tokushoho" },
  ],
},
```

→ Footer から自動的にリンクされる。

### Step 6. Stripe 統合がある場合の利用規約 URL 登録

月額サブスク案件の場合:
1. Stripe Dashboard > Settings > Business > Public details
2. 「利用規約 URL」に `https://本番ドメイン/legal/terms` を登録
3. 「プライバシーポリシー URL」に `https://本番ドメイン/legal/privacy` を登録
4. これにより Stripe Checkout で `consent_collection.terms_of_service: required` を使える

> 📄 詳細: `setup-stripe-subscription` Skill

## 既知の罠 (Pitfalls)

### ❌ 個人事業主の住所をうかつに公開する
- 自宅 = 事業所 のケースで、住所が誰でも見れる状態に
- **特例**: 特商法上、請求があった場合に開示で OK (令和 4 年改正)
- 表記: 「ご請求があった場合は遅滞なく開示します」と明記

### ❌ サブスクの最低契約期間を利用規約に書き忘れる
- 利用規約 第 8 条 + 特商法表記 の両方に明記
- Stripe Dashboard の Terms of Service URL に登録した上で、Checkout で `consent_collection` を使う

### ❌ 中途解約手数料の計算ロジックが規約と実装で乖離
- 規約: 「残月分を一括請求」
- 実装: Webhook で `(remainingMonths × monthlyPrice)` を自動課金
- **金額の出し方を規約と実装で完全一致させる**

### ❌ プライバシーポリシーで GA4 を書き忘れる
- アクセス解析を入れたら、必ずプライバシーポリシーで「Google Analytics 等の Cookie 利用」を開示
- Cookie 取得への同意 (Cookie バナー) も検討

### ❌ 「景品表示法」違反 (Phase H 関連)
- 「初期費用 0 円」等の表記に注釈 (※ 月額必要) を必ず併記
- 利用規約とは別に、サイト本文 (Hero 等) のチェックが必要

### ❌ 「免責事項」が緩すぎる / 厳しすぎる
- 緩すぎ: 損害が発生した時に責任を全部背負う
- 厳しすぎ: 消費者契約法で無効化される
- 推奨: 「過去 12 ヶ月の利用料金」を上限とする

## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| Stripe Checkout でエラー: `Terms of service URL must be set` | Stripe Dashboard > Public details で URL 登録 + `STRIPE_TOS_CONFIGURED=true` に |
| 利用規約に「クライアント名」が空欄のまま | `src/lib/content.ts` の `SITE.name` をセット、ページで `{SITE.name}` を使う |
| 中途解約時にお客様トラブル | 規約と実装の金額計算ロジックを再確認 |
| 個人情報の問合せが来た | プライバシーポリシーに記載した「お問い合わせ窓口」で受付、開示請求対応 |

## 参考: 業種別の補足項目

| 業種 | 利用規約に追加すべき項目 |
|---|---|
| **EC (物販)** | 商品の引渡時期、返品ポリシー、不良品対応 |
| **スクール (継続)** | 受講ルール、コース変更、休講・休会 |
| **コンサル (顧問)** | 守秘義務、成果物の権利帰属 |
| **医療** | 「本サイトは医療行為ではない」の免責 |
